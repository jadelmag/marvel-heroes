# ZARAWEBCHALLENGE

## Uso de Linters y Formatters.

Se ha utilizado oxlint, es una librería que contiene prettier y es compatible con la configuración de eslint.

## Vitest

Para realizar las pruebas unitarias se ha utilizado `vitest`.

## Cypress

Para realizar las pruebas E2E se ha utilizado `cypress`.

### REQUISITOS:

● La aplicación tiene que ser responsive.

### Opcional:

● La aplicación puede estar desplegada.

● Uso de SSR (Es posible usar Next.js).

● Se valorará el uso de variables CSS.
